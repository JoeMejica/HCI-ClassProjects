HLD Introduction

The diagrams below exhibit the components that make up the overall architecture of the website including the use case interaction between the admin/web developer called "admin" and the target audience in this called "user". The prototype and final builds of the website will heavily derive from the high level design in order to maintain design consistency and to maintain a foundation in the development of the website.

Use Case Diagram

As the primary purpose of this project is to create a website to inform the target audience about the Georgia History Museum. uni-directional HCI was decided in order to maintain a reliable "write and read" interaction. Users will be able to interact with the the website dynamically via JavaScript/JQuery and to external links but no form of return communication to "admin" will be utlized.

Web Architecutre

A dedicated server owned by the Georgia History Musuem's IT department will be the primary hosting service for the website's HTML, CSS, and JS. A SPA (single page application) web design will be the primary foundation that the website will be built upon with some webpages utilizing additional HTML pages to contain the needed information. Additionally, Cloudinary CDN service will be utilized to host and manage images/graphics that will appear in the website.
